# ADR-003: 대시보드는 웹 — 데몬이 HTTP API를 서빙하고 모든 클라이언트가 이를 공유한다

- 상태: 승인 (2026-06-06)

## 맥락

TUI(터미널) vs 웹. 요구사항: PC에서 하던 작업을 모바일에서 이어서 확인/투입 가능해야 하고, 최종적으로 홈서버 Docker에서 구동.

## 결정

- 데몬이 **HTTP API**(Hono)를 서빙 — jobs 조회, brief 투입, retry/cancel
- `dashboard/`는 이 API를 쓰는 웹 SPA. Tailscale 경유로 폰/외부에서 접근
- `agentctl` CLI도 **같은 API의 클라이언트** — API가 단일 진실원

## 이유

- TUI는 머신 앞에 있어야 한다. 홈서버+모바일 시나리오와 충돌
- API 우선으로 만들면 CLI/웹/추후 알림봇이 전부 같은 표면을 공유 — 클라이언트별 로직 중복 없음

## 결과

- API 인증: 1차는 Tailscale 네트워크 경계에 위임 (tailnet 밖 비노출), 공개 배포 시 별도 ADR
- 대시보드 구현은 W5부터 시작한다. 구체 스택과 배포 기본값은 ADR-009를 따른다
